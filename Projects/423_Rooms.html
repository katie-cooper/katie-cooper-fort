CSXL Room Reservation System (423 Project)

A scheduling tool designed for the Computer Science Experience Labs at UNC Chapel Hill.

Technical Specifications

About the Project

Fall 2024

Members: Kathleen Fort, Jordan Weinstein, Bennett Rakower, Srikar Sannidhi

Scope: This feature will be created for the UNC CSXL website for UNC computer science students, moderators, and faculty to use

Timeline: This project began October 2024. This project will conclude in December 2024.

Impact: This feature will be available to all students on the CSXL website. It will be used by students who want to reserve study rooms for their study groups. This feature will allow moderators to ensure that rooms are being used appropriately for the size of the group.

Key Personas

  1. Students reserving a room in the XL hallway. Specifically, it serves personas who are working together with their friends.
  2. Students being added to a room reservation to work with their peers
  3. Moderators of the CSXL website that want to restrict room size based on group size and ensure the rooms are being used appropriately

User Stories

  1. As a student with a large group, I want to be able to reserve a large enough room so that we can have a productive space that is adequate.
  2. As a student in a large group, I want my classmate to be able to add me to the reservation so that I can see that reservation on my account and remove myself if necessary.
  3. As a moderator of the CSXL website, I want to be able to ensure that the rooms are being used by appropriate groups so that as many students as possible can utilise the XL resources.

Features

  • View available rooms and timeslots
  • Add additional students to the reservation
  • Delete students as needed from the reservation
  • Be added on someone else's reservation
  • Book, modify, and cancel reservations
  • Admin interface for managing room availability and user permissions
  • Responsive design for mobile and desktop use

Tech Stack

  • Frontend: Angular, Tailwind CSS
  • Backend: API REST Framework
  • Database: PostgreSQL
  • Deployment: Docker & Heroku
